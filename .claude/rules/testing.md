## 테스트 패턴

<!-- 프로젝트에서 사용하는 테스트 패턴을 기술하세요 -->

- **단위 테스트**: 개별 함수/메서드의 동작을 검증
- **통합 테스트**: 모듈 간 상호작용을 검증
- **E2E 테스트**: 사용자 시나리오 전체 흐름을 검증

### 테스트 작성 원칙

- 각 테스트는 독립적으로 실행 가능해야 합니다
- 테스트 데이터는 테스트 내에서 설정하고 정리합니다
- 외부 의존성은 적절히 모킹합니다
- 테스트 설명은 명확하고 구체적으로 작성합니다
