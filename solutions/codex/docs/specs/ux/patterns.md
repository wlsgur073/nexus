## 6. AI Data Butler UX 명세

### 6.1 전체 통합 전략

AI Data Butler는 별도 페이지가 아닌, 각 화면의 맥락에 통합된 Co-pilot으로 동작한다. 기존 `ai-term-suggest.html`의 독립 화면 패턴을 폐지하고, 다음 3가지 형태로 분산 통합한다.

| 통합 형태            | 설명                                         | 적용 화면                                             |
| -------------------- | -------------------------------------------- | ----------------------------------------------------- |
| **인라인 자동완성**  | 입력 필드에서 실시간 유사 표준 추천 드롭다운 | 통합 탐색기 검색, 신규 신청 폼, Command Palette       |
| **사이드 패널 추천** | 폼 우측에 유사 표준 목록 + 매칭 근거 표시    | 신규 표준 신청, 상세 Sheet 편집 모드                  |
| **맥락형 인사이트**  | 현재 화면 데이터에 기반한 AI 분석 결과       | 승인 워크벤치, 검증 대시보드, 거버넌스 포털, 대시보드 |

### 6.2 화면별 AI 통합 포인트

| 화면              | AI 트리거                    | 표시 방식                        | 내용                                      |
| ----------------- | ---------------------------- | -------------------------------- | ----------------------------------------- |
| 통합 표준 탐색기  | 검색어 입력 (디바운스 300ms) | 검색 Input 하단 Command 드롭다운 | 유사 표준명/약어/물리명 자동완성          |
| 통합 표준 탐색기  | 검색 결과 Empty              | Empty State 내 추천 섹션         | "혹시 이것을 찾으셨나요?" 3건             |
| 상세 Sheet (편집) | 편집 모드 진입               | Sheet 내 인라인 힌트             | "이 필드 수정 시 관련 용어 N건 영향"      |
| 상세 Sheet (삭제) | 삭제 신청 시작               | Stepper 내 추천                  | 영향도 자동 분석, 대체 표준 추천          |
| 신규 표준 신청    | 표준명 입력 (디바운스 300ms) | 우측 사이드 패널                 | 유사 표준 3건 + 유사도% + 매칭 근거       |
| 신규 표준 신청    | 80%+ 유사 발견               | 폼 상단 주황 Alert               | 중복 경고 + 기존 표준 링크                |
| 신규 표준 신청    | 영문약어 입력                | Input 하단 힌트                  | AI 추천 약어 (예: "고객유형" → "CUST_TP") |
| 승인 워크벤치     | 신청 상세 조회               | 상세 패널 내 인라인 태그         | 유사 표준 경고, 명명규칙 체크 결과        |
| 승인 워크벤치     | 반려 버튼 클릭               | 사유 Textarea 상단               | 반려 사유 템플릿 추천                     |
| 검증 대시보드     | 페이지 로드                  | 인사이트 카드                    | 위반 패턴 분석 요약, 자동 시정 제안       |
| 검증 상세         | 위반 항목 조회               | 위반 내용 컬럼 인라인            | AI 시정 제안 (예: "payAmt → PAY_AMT")     |
| 거버넌스 포털     | 페이지 로드                  | AI 요약 카드                     | 준수율 추이 분석, 개선 제안               |
| 대시보드 (신청자) | 페이지 로드                  | 플로팅 버튼 Popover              | 맥락 기반 알림 ("피드백 수정 제안 1건")   |
| Command Palette   | Ctrl+K 검색                  | 검색 결과 내                     | 유사 표준 실시간 추천                     |

### 6.3 AI 추천 패널 상세 UX

신규 표준 신청 화면의 우측 사이드 패널 구성:

```markdown
┌────────────────────────────┐
│ 🤖 AI Data Butler │
│ │
│ 유사 표준 3건 발견: │
│ │
│ ┌────────────────────────┐ │
│ │ 1. 고객구분 │ │
│ │ CUST_DIV │ │
│ │ 유사도: ██████░░ 92% │ │
│ │ 상태: ● 기존 │ │
│ │ 유형: 표준단어 │ │
│ │ [상세 보기] [사용하기] │ │
│ └────────────────────────┘ │
│ │
│ ┌────────────────────────┐ │
│ │ 2. 고객유형코드 │ │
│ │ CUST_TP_CD │ │
│ │ 유사도: █████░░░ 85% │ │
│ │ 상태: ● 기존 │ │
│ │ 유형: 표준용어 │ │
│ │ [상세 보기] [사용하기] │ │
│ └────────────────────────┘ │
│ │
│ ── AI 매칭 근거 ── │
│ (선택 항목의 근거 표시) │
│ │
│ 음절 유사도 ████████ 100%│
│ 의미 유사도 ███████░ 95%│
│ 약어 패턴 ██████░░ 88%│
│ │
│ ⚠️ "고객구분"과 92% 유사 │
│ 합니다. 기존 표준을 확인해 │
│ 주세요. │
│ │
└────────────────────────────┘
```

**인터랙션**:

- "상세 보기": 해당 표준의 상세 Sheet 열림 (현재 폼 입력 유지)
- "사용하기": 기존 표준 채택, 신규 신청 폼 닫힘, 해당 표준 상세 Sheet 열림
- 유사도 색상: 80%+ 파랑, 50-80% 주황, 50%- 회색
- 80%+ 유사 시 주황 경고 Alert 자동 표시

### 6.4 AI 데이터 흐름

```markdown
사용자 입력 (검색어/표준명)
│
▼
디바운스 (300ms)
│
▼
GET /api/ai/suggest?keyword={input}&type={type}&limit=3
│
▼
┌─ 결과 있음 ─── 추천 패널/드롭다운 표시
│ │
│ └─ 80%+ 유사 ─── 중복 경고 Alert 표시
│
└─ 결과 없음 ─── "유사 표준 없음. 신규 등록을 진행하세요" 메시지
```

## 7. 공유 UX 패턴

### 7.1 Toast 알림

시스템 전역에서 사용하는 일시적 피드백 메시지.

| 유형    | 색상 | 예시                                     | 자동 닫힘 |
| ------- | ---- | ---------------------------------------- | --------- |
| Success | 녹색 | "변경 신청이 제출되었습니다 (REQ-XXXX)"  | 5초       |
| Info    | 파랑 | "초안이 저장되었습니다"                  | 5초       |
| Warning | 주황 | "유사 표준이 발견되었습니다"             | 수동 닫기 |
| Error   | 적색 | "처리에 실패했습니다. 다시 시도해주세요" | 수동 닫기 |

**위치**: 화면 우측 하단 고정, 최대 3개 스택
**컴포넌트**: shadcn/ui Toast/Sonner

### 7.2 Stepper (단계 진행)

삭제 영향도 평가 등 다단계 프로세스에 사용.

```markdown
[1. 영향 시스템] ─── [2. 영향도 수준] ─── [3. 대체 표준] ─── [4. 삭제 사유]
●완료 ●현재 ○미진행 ○미진행
```

**규칙**:

- 완료 단계: 녹색 체크 아이콘, 클릭 시 이전 단계 복귀 가능
- 현재 단계: 파랑 활성 표시
- 미진행 단계: 회색, 클릭 불가
- "이전" / "다음" 버튼으로 단계 이동
- 마지막 단계에서 "제출" 버튼

### 7.3 Sheet (사이드 패널)

표준 상세 보기, 인라인 편집에 사용하는 우측 슬라이드 패널.

**규격**:

- 너비: 640px (데스크톱), 전체 너비 (모바일)
- 진입: 좌측에서 우측으로 슬라이드 애니메이션
- 닫기: X 버튼, ESC 키, 오버레이 클릭
- 중첩: 최대 2단 (상세 Sheet 위에 확인 Dialog)
- 배경: 반투명 오버레이 (opacity 0.5)

**컴포넌트**: shadcn/ui Sheet (side: right)

### 7.4 Command Palette

전역 검색 + 빠른 네비게이션 + 액션 실행.

**규격**:

- 트리거: Ctrl+K / Cmd+K, 헤더 검색 아이콘
- 위치: 화면 중앙 상단, 오버레이 위
- 너비: 최대 640px
- 섹션: 최근 검색 / 빠른 이동 / 액션 / 표준 검색
- 검색: fuzzy matching, 실시간 결과 갱신
- 키보드: 화살표 키 이동, Enter 선택, ESC 닫기

**컴포넌트**: shadcn/ui Command

### 7.5 알림 센터

실시간 알림 수신 및 확인.

**트리거**: 헤더의 벨 아이콘 (미확인 건수 Badge)
**표시**: 헤더 하단 Popover (최대 높이 400px, 스크롤)

```markdown
┌────────────────────────────┐
│ 알림 [모두 읽음] │
├────────────────────────────┤
│ ● [승인] "고객번호" 승인됨 │
│ 김검토 | 5분 전 │
│ │
│ ● [피드백] "결제수단" 수정요청│
│ 박승인 | 1시간 전 │
│ │
│ ○ [신규신청] 승인 대기 1건 │
│ 이분석 | 어제 │
│ │
│ "전체 알림 보기 →" │
└────────────────────────────┘
```

**알림 유형**:

| 유형           | 대상 역할   | 내용                                             |
| -------------- | ----------- | ------------------------------------------------ |
| 신규 신청 도착 | 승인자      | "새 신청이 도착했습니다: {항목명} ({유형})"      |
| 승인 완료      | 신청자      | "'{항목명}' 신청이 승인되었습니다"               |
| 반려           | 신청자      | "'{항목명}' 신청이 반려되었습니다. 사유: {사유}" |
| 피드백 요청    | 신청자      | "'{항목명}'에 대한 수정 요청이 있습니다"         |
| 검증 완료      | 표준 관리자 | "표준 검증이 완료되었습니다. 위반 {N}건 발견"    |

### 7.6 필터/검색 패턴

모든 목록 화면에서 일관된 필터/검색 UX를 사용한다.

**검색 바 구조**:

```markdown
🔍 [키워드 검색_______________] [필터1 ▾] [필터2 ▾] ... [검색] [초기화]
```

**규칙**:

- 키워드 검색: 디바운스 300ms, 자동 검색 (별도 버튼 불필요)
- 드롭다운 필터: 선택 즉시 결과 갱신
- "초기화" 버튼: 모든 필터를 기본값으로 리셋
- 필터 상태: URL query param에 반영 (뒤로 가기/공유 가능)
- 활성 필터 표시: 필터 영역 하단에 선택된 필터 칩 표시 (제거 가능)

### 7.7 테이블 공통 패턴

| 패턴         | 설명                                                                                                          |
| ------------ | ------------------------------------------------------------------------------------------------------------- |
| 정렬         | 컬럼 헤더 클릭으로 정렬 (ASC ↔ DESC 토글)                                                                     |
| 페이지네이션 | 하단 좌측: "총 N건", 하단 우측: 페이지 네비게이션                                                             |
| 행 클릭      | 상세 보기 Sheet 또는 확장 패널                                                                                |
| 상태 Badge   | 녹색(기존/활성/승인), 주황(신청/피드백), 적색(반려/삭제/높음), 파랑(검토/시정중/중간), 회색(취소/미처리/낮음) |
| Mono 텍스트  | 영문약어, 물리명, 코드, 일시 등 기술 값에 monospace 폰트 적용                                                 |
| 말줄임       | 정의 등 긴 텍스트는 1줄 말줄임 + 호버 시 전체 표시                                                            |
| 체크박스     | 일괄 처리가 필요한 테이블에만 추가 (승인 워크벤치, 검증 상세)                                                 |

### 7.8 상태 Badge 색상 체계

| 색상 | CSS Token             | 적용 상태                                                             |
| ---- | --------------------- | --------------------------------------------------------------------- |
| 녹색 | `--color-success`     | 기존(BASELINE), 활성(ACTIVE), 승인(APPROVED), 완료(RESOLVED), 사용(Y) |
| 주황 | `--color-warning`     | 신청(PENDING), 피드백대기(FEEDBACK), 중간(MEDIUM)                     |
| 적색 | `--color-destructive` | 반려(REJECTED), 삭제(DELETED), 높음(HIGH), 비활성(INACTIVE)           |
| 파랑 | `--color-primary`     | 검토(REVIEW), 시정중(CORRECTING), 베타                                |
| 회색 | `--color-muted`       | 취소(CANCELLED), 미처리(UNRESOLVED), 낮음(LOW), 미사용(N)             |
| 보라 | `--color-accent`      | AI 관련 Badge, Data Butler 라벨                                       |

### 7.9 확인 Dialog 패턴

삭제, 취소, 신청 제출 등 중요 액션 전에 표시하는 확인 대화상자.

```markdown
┌────────────────────────────┐
│ ⚠️ 신청을 취소하시겠습니까? │
│ │
│ 취소된 건은 복구할 수 없으며,│
│ 감사 추적에 기록됩니다. │
│ │
│ [아니오] [취소 확인] │
└────────────────────────────┘
```

**규칙**:

- 파괴적 액션(삭제, 취소)은 적색 확인 버튼
- 일반 확인(신청 제출)은 파랑/녹색 확인 버튼
- ESC 키로 닫기 가능
- "아니오" 버튼이 항상 좌측 (실수 방지)

## 8. 접근성 & 반응형 고려사항

### 8.1 접근성 (a11y)

| 항목              | 기준                                 | 구현 방법                                         |
| ----------------- | ------------------------------------ | ------------------------------------------------- |
| 키보드 네비게이션 | 모든 인터랙티브 요소에 Tab 접근 가능 | `tabindex`, `focus-visible` 스타일                |
| 스크린 리더       | ARIA 라벨, role 속성 적용            | `aria-label`, `aria-describedby`, `role`          |
| 색상 대비         | WCAG 2.1 AA 기준 (4.5:1 이상)        | Tailwind CSS v4 테마 토큰 색상 검증               |
| 포커스 표시       | 포커스 링 시각적 표시                | `ring-2 ring-offset-2` Tailwind 유틸리티          |
| Badge 색상        | 색맹 사용자를 위한 텍스트 라벨 병행  | Badge에 항상 텍스트 라벨 포함 (색상만 의존 안 함) |
| 폼 에러           | 에러 메시지를 필드와 연결            | `aria-invalid`, `aria-errormessage`               |
| 모달/Sheet        | 포커스 트랩, ESC 닫기                | shadcn/ui 기본 제공                               |
| 테이블            | 헤더 셀에 `scope` 속성               | `<th scope="col">`                                |

### 8.2 반응형 브레이크포인트

Nexus PlatformShell의 반응형 브레이크포인트를 따른다.

| 브레이크포인트 | 너비         | 레이아웃 변경                                  |
| -------------- | ------------ | ---------------------------------------------- |
| Desktop (기본) | 1280px+      | 사이드바 260px + 메인 콘텐츠                   |
| Tablet         | 768px-1279px | 사이드바 접힘 (아이콘만), 햄버거 메뉴          |
| Mobile         | ~767px       | 사이드바 숨김 (모바일 드로어), Sheet 전체 너비 |

**반응형 처리 세부사항**:

| 컴포넌트                  | Desktop             | Tablet               | Mobile                           |
| ------------------------- | ------------------- | -------------------- | -------------------------------- |
| 사이드바                  | 260px 고정          | 아이콘 모드 (60px)   | 숨김, 햄버거 드로어              |
| Stat 카드                 | 4-column grid       | 2-column grid        | 1-column 스택                    |
| 좌우 분할 (승인 워크벤치) | 45/55 분할          | 40/60 분할           | 1-column 스택 (목록 → 상세 전환) |
| 좌우 분할 (공통코드 관리) | 320px/나머지        | 접힘/전체 전환       | 1-column 스택                    |
| Sheet                     | 640px 우측 슬라이드 | 480px                | 전체 너비                        |
| 테이블                    | 전체 컬럼           | 핵심 컬럼만 (스크롤) | 카드형 리스트 전환               |
| Command Palette           | 640px 중앙          | 480px                | 전체 너비 - 패딩                 |
| 차트                      | 기본 크기           | 축소                 | 스크롤 또는 심플 수치            |

### 8.3 다크 모드 지원

Nexus PlatformShell의 `next-themes` 기반 다크 모드를 활용한다.

- **기본값**: 시스템 모드 (OS 설정 따름)
- **전환**: 헤더의 ThemeToggle 컴포넌트
- **구현**: Tailwind CSS v4 `@theme inline` 블록에 정의된 CSS 변수 기반
- **Badge 색상**: 다크 모드에서도 가독성 유지 (밝기 조정)
- **차트 색상**: 다크 모드 전용 팔레트 제공
