## 5. 화면 명세 — 핵심 화면

### 5.1 로그인

**목적**: Codex 시스템 접근을 위한 사용자 인증
**진입 경로**: 비인증 상태에서 모든 URL 접근 시 자동 리다이렉트
**경로**: `/solutions/codex/login`

**레이아웃**: Split-screen (PlatformShell 미적용, 사이드바 없음)

```markdown
┌───────────────────────────┬────────────────────────┐
│ │ │
│ 좌측 (55%) │ 우측 (45%) │
│ 브랜딩 영역 │ 로그인 폼 │
│ │ │
│ ┌────────────────┐ │ ┌────────────────┐ │
│ │ Codex 로고 │ │ │ 사용자 아이디 │ │
│ │ Enterprise │ │ │ [__________] │ │
│ │ Standard │ │ │ │ │
│ │ Platform │ │ │ 비밀번호 │ │
│ └────────────────┘ │ │ [__________] │ │
│ │ │ │ │
│ ┌──┐ ┌──┐ ┌──┐ │ │ [로그인 버튼] │ │
│ │통제│ │감사│ │역할│ │ │ │ │
│ │거버│ │추적│ │기반│ │ │ 환경설정 | 지원 │ │
│ └──┘ └──┘ └──┘ │ └────────────────┘ │
│ │ │
└───────────────────────────┴────────────────────────┘
```

**주요 컴포넌트 & 인터랙션**:

| 컴포넌트  | shadcn/ui 매핑              | 설명                                       |
| --------- | --------------------------- | ------------------------------------------ |
| 로그인 폼 | `Card` + `Input` + `Button` | 사용자 아이디/비밀번호 입력                |
| 피처 카드 | `Card` (3-column grid)      | 통제된 거버넌스, 감사 추적, 역할 기반 접근 |
| 하단 링크 | 텍스트 링크                 | 환경설정(DB 설정), 지원 요청               |

**상태별 UI**:

| 상태              | 표시                                                              |
| ----------------- | ----------------------------------------------------------------- |
| Loading           | 로그인 버튼 Spinner + 비활성화                                    |
| Error (인증 실패) | 폼 상단 적색 Alert: "아이디 또는 비밀번호가 올바르지 않습니다"    |
| Error (서버 오류) | 적색 Alert: "서버 연결에 실패했습니다. 잠시 후 다시 시도해주세요" |

**역할별 차이점**: 없음 (로그인 전이므로 동일)

**AI Data Butler**: 미적용 (로그인 전)

### 5.2 신청자 대시보드

**목적**: 신청자 관점에서 본인 신청 현황과 표준 시스템 요약을 한눈에 파악
**진입 경로**: 로그인 후 자동 이동 (역할이 신청자/조회전용인 경우)
**경로**: `/solutions/codex` (역할별 뷰 자동 전환)

**레이아웃**: PlatformShell > Main Content

```markdown
┌──────────────────────────────────────────────────────┐
│ Topbar: "대시보드" [알림 🔔(3)] [Ctrl+K 검색] │
```

**주요 컴포넌트 & 인터랙션**:

| 컴포넌트           | shadcn/ui 매핑         | 설명                                                      |
| ------------------ | ---------------------- | --------------------------------------------------------- |
| 내 신청 Stat 카드  | `Card` (4-column grid) | 전체/승인대기/피드백/반려 건수. 클릭 시 아래 목록 필터링  |
| 내 최근 신청 목록  | `Card` > 커스텀 리스트 | 최근 5건 표시, 상태별 액션 버튼 (취소, 수정/재신청, 상세) |
| 시스템 현황 카드   | `Card` (3-row)         | 표준단어/도메인/용어 총 건수, 월대비 증감                 |
| 등록 추이 차트     | `Card` > 바 차트       | 6개월 등록 추이 (recharts 또는 유사 라이브러리)           |
| 최근 활동 타임라인 | `Card` > 타임라인      | 최근 5건, 시간순 정렬, 액션 배지                          |

**상태별 UI**:

| 상태              | 표시                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| Loading           | Stat 카드 Skeleton, 목록 Skeleton (3행)                              |
| Empty (신청 없음) | "아직 신청한 표준이 없습니다. [신규 표준 신청하기 →]" 일러스트 + CTA |
| Error             | "대시보드 데이터를 불러오지 못했습니다. [다시 시도]" Alert           |

**역할별 차이점**:

- **조회전용 사용자**: 내 신청 영역 숨김, 시스템 현황 + 최근 활동만 표시
- **표준 관리자**: 내 신청 영역 + 거버넌스 요약 위젯 추가 (준수율 게이지 미니)

**AI Data Butler**: 대시보드 우측 하단에 플로팅 버튼. 클릭 시 "오늘 승인된 표준 3건", "피드백 대기 중인 신청 1건에 대한 수정 제안" 등 맥락 기반 알림 표시.

### 5.4 통합 표준 탐색기

**목적**: 표준단어, 표준도메인, 표준용어를 하나의 화면에서 탭으로 전환하며 검색/탐색
**진입 경로**: 사이드바 "표준 탐색기", Command Palette 검색, 대시보드 링크
**경로**: `/solutions/codex/standards`

**레이아웃**: PlatformShell > Main Content (탭 + 테이블 + 우측 Sheet)

```markdown
┌──────────────────────────────────────────────────────┐
│ Topbar: "통합 표준 탐색기" [신규 신청 +] [Ctrl+K] │
├──────────────────────────────────────────────────────┤
│ │
│ ┌─────────┬─────────┬──────────┐ │

│ │
```

**주요 컴포넌트 & 인터랙션**:

| 컴포넌트       | shadcn/ui 매핑        | 설명                                                              |
| -------------- | --------------------- | ----------------------------------------------------------------- |
| 유형 탭        | `Tabs`                | 표준용어/표준단어/표준도메인 전환. 각 탭에 총 건수 Badge          |
| 검색 바        | `Input` (search type) | 키워드 입력. 타이핑 시 AI 자동완성 드롭다운                       |
| 필터 드롭다운  | `Select` (복수)       | 상태 (전체/기존/신청/삭제), 도메인유형, 인포타입 (탭별 필터 상이) |
| 결과 테이블    | `Table`               | 탭별 컬럼 구성 변경 (아래 상세)                                   |
| 행 클릭        | → `Sheet` 열림        | 해당 표준의 상세 보기 Sheet (5.5) 우측 슬라이드                   |
| 페이지네이션   | 커스텀 Pagination     | 10건/페이지                                                       |
| 신규 신청 버튼 | `Button` (primary)    | Topbar 우측, 신규 표준 신청 화면으로 이동                         |

**탭별 테이블 컬럼**:

| 표준용어 탭   | 표준단어 탭     | 표준도메인 탭     |
| ------------- | --------------- | ----------------- |
| No            | No              | No                |
| 표준용어      | 표준단어        | 도메인명          |
| 물리명 (mono) | 영문약어 (mono) | 도메인유형        |
| 도메인유형    | 영문명          | 데이터타입 (mono) |
| 인포타입      | 정의 (말줄임)   | 데이터길이 (mono) |
| 정의 (말줄임) | 도메인유형      | 정의 (말줄임)     |
| 상태 (Badge)  | 상태 (Badge)    | 상태 (Badge)      |
| 등록일        | 등록일          | 등록일            |

**상태별 UI**:

| 상태                   | 표시                                                                                         |
| ---------------------- | -------------------------------------------------------------------------------------------- |
| Loading                | 테이블 Skeleton (5행), 탭/필터 비활성화                                                      |
| Empty (검색 결과 없음) | "검색 결과가 없습니다" 아이콘 + "AI Data Butler에게 물어보기" CTA + "신규 표준 신청하기" CTA |
| Empty (데이터 없음)    | "등록된 {유형명}이 없습니다. [신규 신청하기 →]"                                              |
| Error                  | Alert: "데이터를 불러오지 못했습니다. [다시 시도]"                                           |

**역할별 차이점**:

- **조회전용**: "신규 신청" 버튼 숨김, 행 클릭 시 상세 Sheet 읽기 전용
- **신청자/표준 관리자**: 전체 기능 사용 가능
- **승인자**: 검색/조회 가능, 상세 Sheet에서 신청 버튼 숨김

**AI Data Butler 통합**:

- **검색 자동완성**: 검색어 입력 시 AI가 유사 표준명/영문약어/물리명 실시간 추천 (Command 드롭다운)
- **Empty State 추천**: 검색 결과 없을 때 "혹시 이것을 찾으셨나요?" 3건 추천
- **유사 표준 경고**: 신규 신청 버튼 클릭 전, 검색어와 유사한 기존 표준이 있으면 인라인 안내

### 5.5 표준 상세 보기 & 인라인 거버넌스

**목적**: 통합 표준 탐색기에서 행 클릭 시 열리는 상세 보기. 인라인 편집으로 변경/삭제 신청까지 한 화면에서 처리
**진입 경로**: 통합 표준 탐색기에서 행 클릭, Command Palette 검색 결과 클릭, 다른 화면의 표준명 링크 클릭
**경로**: URL 변경 없이 Sheet 오버레이 (query param `?detail={id}&type={word|domain|term}`)

**레이아웃**: 우측 슬라이드 Sheet (너비 640px, 탐색기 위에 오버레이)

```markdown
┌──────────────────────┬───────────────────────────────┐
│ │ Sheet: 고객번호 (CUST_NO) ✕ │
│ (탐색기 배경, │ │
│ 어두워짐) │ [읽기 모드] ↔ [편집 모드] │
│ │ │
│ │ ── 기본 정보 ── │
│ │ 표준용어 고객번호 │
│ │ 물리명 CUST_NO │
│ │ 도메인유형 문자열 │
│ │ 인포타입 코드 │
│ │ 정의 고객을 식별하는 │
│ │ 고유 번호 │
│ │ 상태 ● 기존 │
│ │ 등록일 2025-08-15 │
│ │ 최종수정일 2026-01-20 │
│ │ │
│ │ ── 관련 표준 ── │
│ │ 구성 단어: │
│ │ 고객(CUST) ● 기존 │
│ │ 번호(NO) ● 기존 │
│ │ 사용 도메인: │
│ │ 문자열 / VARCHAR(20) │
│ │ │
│ │ ── 변경 이력 ── │
│ │ ● 2026-01-20 변경승인 박승인 │
│ │ ● 2025-08-15 신규승인 김관리 │
│ │ │
│ │ ┌────────────────────────┐ │
│ │ │ [변경 신청] [삭제 신청] │ │
│ │ └────────────────────────┘ │
│ │ │
└──────────────────────┴───────────────────────────────┘
```

**인라인 편집 모드 (변경 신청)**:

```markdown
┌───────────────────────────────┐
│ Sheet: 고객번호 — 편집 모드 ✕ │
│ │
│ ── 기본 정보 (편집 가능) ── │
│ 표준용어 [고객번호_________] │
│ 물리명 CUST_NO (readonly) │
│ 도메인유형 [문자열 ▾] │
│ 인포타입 [코드 ▾] │
│ 정의 [고객을 식별하는 │
│ 고유 번호______] │
│ │
│ ── 변경 미리보기 ── │
│ ┌────────┬────────┬────────┐ │
│ │ 항목 │ 현재값 │ 변경값 │ │
│ ├────────┼────────┼────────┤ │
│ │ 정의 │ 고객을 │ 고객을 │ │
│ │ │ 식별하는│ 고유하게│ │
│ │ │ 고유번호│ 식별하는│ │
│ │ │ │ 관리번호│ │
│ └────────┴────────┴────────┘ │
│ \* 변경된 필드만 표시 │
│ │
│ ── 신청 정보 ── │
│ 신청사유 [정의 명확화를 위해 │
│ 변경 신청합니다___] │
│ │
│ ┌────────────────────────┐ │
│ │ [초안 저장] [변경 신청 제출]│ │
│ └────────────────────────┘ │
│ │
└───────────────────────────────┘
```

**삭제 신청 모드 (Sheet 내 스텝 전환)**:

```markdown
Sheet 내에서 Stepper로 삭제 영향도 평가 진행:

Step 1/4: 영향 받는 시스템
☑ 운영DB ☑ DW ☐ 보고서 ☐ API ☐ 외부연동 ☐ 기타
기타 영역: [________________]

Step 2/4: 영향도 수준
○ 높음 ● 중간 ○ 낮음
설명\*: [영향도 설명________________]

Step 3/4: 대체 표준 제안
대체 표준: [검색____________] (AI 추천 인라인)
마이그레이션 계획: [________________]

Step 4/4: 삭제 사유
```

**주요 컴포넌트 & 인터랙션**:

| 컴포넌트      | shadcn/ui 매핑               | 설명                                                                |
| ------------- | ---------------------------- | ------------------------------------------------------------------- |
| 상세 패널     | `Sheet` (side: right, 640px) | 우측 슬라이드. ESC 또는 오버레이 클릭으로 닫기                      |
| 모드 전환     | `Tabs` 또는 토글             | 읽기 모드 ↔ 편집 모드 전환                                          |
| 기본 정보     | `Card` 내 key-value 레이아웃 | 읽기 모드: 텍스트 표시, 편집 모드: Input/Select/Textarea            |
| 관련 표준     | `Card` > 미니 `Table`        | 구성 단어, 사용 도메인 정보                                         |
| 변경 이력     | `Card` > 타임라인            | 시간순 정렬, 최대 10건                                              |
| 변경 미리보기 | `Table` (3-column diff)      | 변경된 필드만 표시, 변경값 하이라이트                               |
| 삭제 Stepper  | 커스텀 Stepper               | 4단계 영향도 평가                                                   |
| 인라인 코멘트 | 필드별 말풍선 아이콘         | 편집 모드에서 각 필드 옆 코멘트 아이콘. 클릭 시 코멘트 입력 Popover |
| 초안 저장     | `Button` (secondary)         | Draft 상태로 저장, 나중에 제출 가능                                 |
| 신청 제출     | `Button` (primary)           | 확인 Dialog 후 신청 생성                                            |

**상태별 UI**:

| 상태                   | 표시                                                                         |
| ---------------------- | ---------------------------------------------------------------------------- |
| Loading                | Sheet 내 Skeleton (정보 카드 + 이력)                                         |
| Error (상세 로드 실패) | Sheet 내 Alert: "정보를 불러오지 못했습니다" + 닫기                          |
| 편집 중 + 저장         | 버튼 Spinner, 폼 비활성화                                                    |
| 신청 제출 성공         | Sheet 닫힘 + Toast: "변경 신청이 제출되었습니다 (REQ-XXXX)"                  |
| 초안 저장 성공         | Toast: "초안이 저장되었습니다. 신규 신청 화면에서 이어서 작성할 수 있습니다" |

**역할별 차이점**:

- **조회전용**: 편집 모드 전환 불가, 변경/삭제 신청 버튼 숨김
- **승인자**: 변경/삭제 신청 버튼 숨김 (승인자는 검토만)
- **신청자/표준 관리자**: 전체 기능 사용 가능

**AI Data Butler 통합**:

- **편집 모드 진입 시**: "이 필드를 수정하면 관련 용어 N건에 영향이 있습니다" 안내
- **정의 필드 편집 시**: AI가 기존 유사 정의와 비교하여 개선 제안 표시 (인라인 힌트)
- **삭제 신청 시**: AI가 영향도를 자동 분석하여 "영향 받는 시스템 추천 체크" 제안
