## 5. 화면 명세 — 거버넌스 및 관리 화면

### 5.6 신규 표준 신청

**목적**: 존재하지 않는 새로운 표준(단어/도메인/용어)을 신규 등록 신청
**진입 경로**: 사이드바 "신규 신청", 탐색기 "신규 신청" 버튼, AI 추천의 "그래도 신규 신청" 링크
**경로**: `/solutions/codex/standards/new`

**레이아웃**: PlatformShell > Main Content (전체 페이지 폼)


**초안 시스템 (Draft)**:

- "초안 저장" 클릭 시 로컬/서버에 Draft 상태로 저장
- Topbar의 "초안 목록 (N)" 클릭 → Popover로 저장된 초안 목록 표시
- 초안 선택 시 폼에 자동 입력, 이어서 작성 가능
- 초안은 최대 10건까지 저장, 30일 후 자동 삭제
- 초안 목록에서 개별 삭제 가능

**주요 컴포넌트 & 인터랙션**:

| 컴포넌트     | shadcn/ui 매핑                       | 설명                                                        |
| ------------ | ------------------------------------ | ----------------------------------------------------------- |
| 유형 선택    | 라디오 카드 (3개)                    | 표준단어/표준도메인/표준용어 선택. 선택에 따라 폼 필드 변경 |
| 신청 폼      | `Card` > `Input`/`Select`/`Textarea` | 유형별 필수/선택 필드 구성                                  |
| AI 추천 패널 | `Card` (우측 고정)                   | 폼 입력 시 실시간 유사 표준 추천                            |
| 중복 경고    | Alert (주황)                         | 80%+ 유사 표준 발견 시 표시                                 |
| 초안 저장    | `Button` (secondary)                 | Draft로 저장                                                |
| 신청 제출    | `Button` (primary)                   | 확인 Dialog 후 신청 생성                                    |

| 도메인유형      |    O     |    O\*     |     O\*      |
| 데이터타입      |    -     |    O\*     |      -       |
| 데이터길이      |    -     |     O      |      -       |

**상태별 UI**:

| 상태                 | 표시                                                                      |
| -------------------- | ------------------------------------------------------------------------- |
| Loading (AI 추천)    | AI 패널 Skeleton (3행)                                                    |
| Empty (AI 추천 없음) | "유사한 기존 표준이 없습니다. 새로운 표준을 등록하세요!"                  |
| 유효성 검증 실패     | 필수 필드 적색 border + 하단 오류 메시지                                  |
| 제출 중              | 버튼 Spinner + 폼 비활성화                                                |
| 제출 성공            | Dialog: "신청이 제출되었습니다 (REQ-XXXX). [대시보드로 이동] [계속 신청]" |
| 초안 저장 성공       | Toast: "초안이 저장되었습니다"                                            |

**역할별 차이점**: 이 화면은 신청 권한이 있는 역할에게만 접근 가능 (시스템관리자, 표준 관리자, 신청자).

**AI Data Butler 통합**:

- **실시간 유사 표준 추천**: 표준명 필드 입력 시 디바운스(300ms) 후 유사 표준 검색, 우측 패널 갱신
- **중복 경고**: 80%+ 유사 기존 표준 발견 시 주황 Alert, "사용하기" 클릭 시 해당 상세 Sheet 열림
- **영문약어 자동완성**: AI가 표준명으로부터 영문약어 추천 (예: "고객유형" → "CUST_TP")
- **정의 자동 생성**: 표준명과 도메인유형 기반으로 AI가 정의 초안 제안 (수락/수정 가능)

| 경과일 강조      | 색상 표시                         | 7일+ 적색, 3-7일 주황, 3일 미만 기본                          |
| 우측 상세        | 리사이즈 가능 패널                | 신청 개요, 변경 비교, 사유, 코멘트, 이력, 처리                |
| 변경 비교 테이블 | `Table` (3-column diff)           | 현재값/변경요청값 비교, 변경 셀 하이라이트                    |
| 인라인 코멘트    | Popover / 커스텀                  | 특정 필드에 대한 의견 작성, 리뷰 시 활용                      |
| 처리 버튼        | `Button` (3종)                    | 승인(primary), 반려(destructive), 검토요청(secondary)         |
| 일괄 처리        | Topbar `Button`                   | 체크박스 선택된 항목 일괄 승인/반려 + 사유 입력 Dialog        |

**핵심 비즈니스 규칙**:

- 본인 신청 건은 목록에서 자동 제외 (신청자/승인자 분리 원칙)
- 모든 처리에 처리사유 입력 필수
- 삭제 신청 건은 영향도 평가 내용이 상세 패널에 추가 표시

**상태별 UI**:

| 상태                 | 표시                                                                                |
| -------------------- | ----------------------------------------------------------------------------------- |
| Loading              | 좌측 리스트 Skeleton, 우측 패널 Skeleton                                            |
| Empty (대기 건 없음) | "처리할 승인 건이 없습니다" 아이콘 + 메시지 (좌측 전체)                             |
| 우측 미선택          | "좌측에서 신청 건을 선택하세요" 안내 텍스트                                         |
| 처리 중              | 버튼 Spinner + 비활성화                                                             |
| 처리 성공            | Toast: "{처리유형} 처리되었습니다" + 좌측 목록에서 해당 건 제거 + 다음 건 자동 선택 |
| Error                | Alert: "처리에 실패했습니다. [다시 시도]"                                           |

**역할별 차이점**: 이 화면은 시스템관리자와 검토/승인자에게만 접근 가능.

**AI Data Butler 통합**:

- **유사 표준 경고**: 신규 신청 건에 대해 "기존 '고객유형코드'와 85% 유사합니다" 인라인 안내
- **승인 추천**: AI가 신청 내용을 분석하여 "명명 규칙 준수 확인됨", "필수 항목 충족" 등 자동 체크리스트 표시
- **반려 사유 자동 완성**: 반려 버튼 클릭 시 AI가 빈번한 반려 사유 템플릿 추천
